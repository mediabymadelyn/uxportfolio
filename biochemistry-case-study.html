Case Study · Spring 2025

Oxy Biochemistry Database

A full-stack lab inventory platform that made biological sample tracking faster, clearer, and more reliable for the Occidental College Biochemistry Department.

Back to Work

Overview

I collaborated with Professor Vikram and the Occidental College Biochemistry Department to design and build a centralized system for tracking laboratory strains. The previous process relied on handwritten labels and scattered spreadsheets, making retrieval and coordination difficult.

I focused on UI/UX design, frontend implementation, and documentation while collaborating with teammates on backend API and database architecture.

Problem

  • No centralized inventory system for strains.
  • Samples spread across multiple labs with inconsistent labeling.
  • No quick way to trace room, freezer, shelf, rack, box, and tube location.
  • No barcode-backed ID flow for fast lookup.
  • Different organizational methods across faculty created friction and confusion.

My Role

  • Designed core wireframes for search, sample detail, and add/edit workflows.
  • Defined typography, color, and layout system aligned with Oxy visual conventions.
  • Implemented key React frontend components.
  • Worked with backend teammates to align API responses with UI flow needs.
  • Documented architecture and user-flow decisions for team handoff and iteration.

Design & User Flow

Search

  • General search across all sample fields.
  • Location search matching Room → Freezer → Shelf → Rack → Box → Tube hierarchy.
  • Filter-based queries by professor, marker, date, and additional metadata.
  • View All mode for full inventory scanning.

Sample Detail

  • Displays label, sample name, professor, markers, organism type, and notes.
  • Includes edit, delete, and barcode generation actions.

Add/Edit Workflow

  • Structured fields reduce user error and enforce storage hierarchy consistency.
  • Validation checks prevent tube-location conflicts.

Admin Dashboard

  • Supports adding and managing locations from room level down to box level.

Technical Overview

Frontend

  • React.js UI with dynamic search and sample-detail views.
  • Responsive layouts designed for real lab environments.

Backend

  • Node.js + Express REST API.
  • Validation, error handling, and CORS integration.
  • Environment-variable based configuration.

Database

  • MySQL hosted on AWS RDS.
  • Hierarchical storage schema from room to tube level.
  • Centralized unique IDs used for barcode generation.

Reflection

This project strengthened how I design for a technical scientific audience while balancing usability and data integrity. If I continued this work, I would focus on stronger mobile optimization, expanded user personalization, and batch barcode scanning to speed up inventory workflows.
